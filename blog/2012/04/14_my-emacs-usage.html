Emacs --- 能够显示地图的编辑器？

  •  联合国大学 (UNU)
  • 强迫症 (OCD) 者的计算机使用表现 

今天转移到了 Windows 7 64-bit 下做点事情，在使用 Emacs 时出现了许多问题：首先，程 序打开失去响应，后来发现是 NTFS 文件系统出错，重启系统进行硬盘检查后排除；再后来 发现我的 org-mode 设置中的 org-contacts 工作不正常，出现如下错误： 

File mode specification error: (wrong-type-argument number-or-marker-p nil) [4 times]
Bad sexp at line 27 in d:/Exaos/Workspace/exaos/daily/myorg/contacts.org: (org-contacts-anniversaries)
Preparing diary...done
Preparing diary...done
Bad sexp at line 27 in d:/Exaos/Workspace/exaos/daily/myorg/contacts.org: (org-contacts-anniversaries)
...

搜索了一翻，仍然没有解决这个问题。首先， org-contacts 已经正确加载了，但在我的 联系人列表中有一些人的生日要早于 1970 年，这个在 32-位的 Emacs 上似乎无法解决。 （参见 Stack Overflow 上的相关提问，其中有我的注释。）

在调试这个问题的过程，我发现新版的 org-mode 中自带的 org-contacts 新添了一个 功能：在 google 地图上显示联系人的地址。（当然，这个地图是在 Emacs 中显示的哦。） 你有想过在编辑器中显示 Google 地图吗？ Emacs 就可以，不过，能够显示地图的编辑器 还能算是单纯的编辑器吗？所以，有人说， Emacs 不应该叫编辑器，应该叫基于编辑器的操 作系统。从这种调侃中可见这个软件确实有点牛气冲天。不过，也许会有人反驳说：也有其 它的编辑器能够显示地图，比如某某 IDE 之类的。我想说的是，那是在其它的软件中嵌入编 辑器，而 Emacs 是在编辑器中嵌入其它的软件，这其中的微妙差异还需要仔细体会。

关于 google-maps.el, 请参见：

  • http://julien.danjou.info/software/google-maps.el
  • http://git.naquadah.org/?p=google-maps.git
  • http://julien.danjou.info/blog/2010/google-maps.el

但 elisp 本身也有许多缺点而被人诟病。因此，有人希望采用其它语言来重新实现 Emacs 类似的功能，比如：

  • 使用 Haskell 的 Yi (義) 编辑器
  • 使用 Common Lisp 的 Climacs
  • 还有人在计划着使用 Scheme 的方言 GNU Guile 来重新实现 Emacs, 这个有点值得期待哟 ---
    • http://lists.gnu.org/archive/html/emacs-devel/2010-04/msg00665.html
    • http://lists.gnu.org/archive/html/emacs-devel/2012-04/msg00276.html
    • http://www.google-melange.com/gsoc/proposal/review/google/gsoc2012/bpt/23002

带有 Lisp 方言编程环境的 Emacs 编辑器以其深厚的历史感、实用性和强大的扩展功能，在 编辑器程序中难有出有右者。作为非程序员的程序开发者，我用它主要完成一些如下的工作：

  1. 写文档 (LaTeX)
  2. 写程序 (C/C++, Python, Shell, Lua, …)
  3. 写博客 (org-mode + o-blog)
  4. 写日记 (org-mode: Capture-Refile-Archive)
  5. 读文档 (info + woman)
  6. 继续开发中哦 :-D
