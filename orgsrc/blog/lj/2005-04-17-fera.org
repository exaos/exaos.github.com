#+TITLE: Kmax+CAMAC 系统使用 FERA 方式采集的串谱问题
#+AUTHOR: 心蛛
#+DATE: [2005-04-17 03:33:00]

硬件组合：
  - Ortec AD413A （若干）+ FERA 驱动器和存储器

现象：
  - 多个 ADC 协同工作时会出现几个通道之间的谱相互串（Channels feed each other）
    的现象。

原因：多参数测量中需要几个 ADC 协调工作，而每个 ADC 的转换时间有差异，因此产生
REQ 请求的反应时间也各不相同，有效信号少的 ADC 产生 REQ 响应比较快，而此时有效信
号少的 ADC 还没有转换完成，FERA 控制器就开始读取数据，这会导致读数混乱，引起串谱。
（？）

解决办法：增加 REQ 的延时，等所有的 ADC 都转换完毕再进行读数。AD413 的延时应该 >
24μS。LRS4301 的最大 REQ 延时为 500nS，因此不适宜多个 ADC 同时工作。CMC203 是集
FERA Controller + Memory + Histogrammer 于一体的 CAMAC 器件，可以方便地进行 REQ
等参数的调节，因此能够满足多个 ADC 协同工作的需要。

CMC203 的逻辑器件：3 块 CPLD 和 2 块 FPGA
